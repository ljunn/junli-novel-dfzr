# 质量门

所有产物交付前，至少过一遍四维评估：情节、人物、世界、情感。

## 一、通用四维评估

### 情节维
- 当前产物是否清楚回答“谁想要什么、被什么阻止、失败代价是什么”
- 是否存在具体推进，而不是原地解释
- 是否至少制造了一个新的未决问题或升级了旧问题

### 人物维
- 关键人物的行为是否符合既有欲望、缺口和压力
- 角色之间是否有具体关系张力，而不是站位说明
- 角色有没有通过选择、代价或误判留下痕迹

### 世界维
- 世界规则是否能解释当前冲突
- 资源、制度、禁忌或历史是否真正参与推动剧情
- 没有为了方便推进临时改法则

### 情感维
- 读者应追的情绪是清楚的
- 受压后是否有补偿基础、回应或收益
- 没有把情绪写成空泛感悟

## 二、章节专用门

- 前 20% 有钩子
- 本章有不可删除的核心事件
- 本章至少推进一条主线、支线、关系线或真相线
- 本章至少回应一个旧悬念
- 本章至少给一次回报
- 章末停在未决压力上，不停在总结上

## 三、设定 / 大纲专用门

- 设定能直接长出冲突和代价
- 大纲能看出“终点站 -> 起爆事件 -> 连锁反应 -> 阶段锚点”
- 没有只有设定名词、没有剧情使用面的问题
- 世界、人物、冲突三者已经互相咬合

## 四、返修专用门

- 只修了授权范围内的问题
- 去味后没有把信息、爽点、钩子洗掉
- 如果改了结尾，追更驱动力更强而不是更弱
- 如果改了对白，人物声音更分化而不是更统一

## 五、语言护栏

- 少抽象心理命名，多动作、环境、停顿、后果
- 少解释腔和总结腔
- 少连续同长度句群
- 少成串 AI 套语
- 同一段不钻两个人脑内

## 六、发现问题时的回退顺序

1. 先查道：是不是读者承诺没立住
2. 再查法：是不是题材、POV、平台、逻辑规则失守
3. 再查术：是不是流程跳步，没做意图卡、场景卡或分层返修
4. 最后才改器：重写产物本身
